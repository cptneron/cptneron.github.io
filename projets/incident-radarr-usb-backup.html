Diagnostic

Analyse des causes racines

Cause #1 — Volume Radarr non monté

Lors d'une recréation du conteneur Radarr via Portainer, le volume /movies n'a pas été remapé. Radarr a démarré sans accès à sa bibliothèque et a interprété l'absence de fichiers comme une suppression physique :

        Rescan will not update movies as a failsafe.

Cause #2 — Job USB Backup OMV avec rsync --delete

OMV avait un job USB Backup configuré sur le disque sda1 (le disque de stockage films), traité comme disque de destination de backup. À chaque montage, le service synchronisait une source quasi-vide vers sda1 avec l'option --delete, effaçant tout ce qui n'était pas dans la source :

BASH/var/lib/openmediavault/usbbackup.d/rsync-090cffa0
# Synchronise directories...
rsync --verbose --archive --delete \
  "/srv/.../sdb1/Films/" \
  "/srv/d1d822bcebf0.../" # = sda1, le disque films !
⚠️ Erreur de configuration sda1 était simultanément le disque de stockage films ET la destination d'un job USB Backup. À chaque démarrage du NAS, le service systemd effaçait silencieusement tous les fichiers non présents dans la source de backup.

Résolution

Actions correctives

1. Désactivation du service USB Backup

BASHDésactivation définitive
$ systemctl disable openmediavault-usbbackup-d1d822bcebf0cd3220230fd9527c2f92
Removed "/etc/systemd/system/dev-disk-by\x2duuid-b76c9ccb.device.wants/..."
$ systemctl daemon-reload

2. Correction du docker-compose Radarr

Suppression du double montage confus (/movies et /movies2), conservation d'un seul dossier racine clair :

YAMLdocker-compose.yml — Radarr (corrigé)
services:
  radarr:
    image: lscr.io/linuxserver/radarr:latest
    volumes:
      - /srv/.../Configs/radarr:/config
      - /srv/dev-disk-by-uuid-b76c9ccb...:/movies  # MOVIES2, 3.6T
      - /srv/.../Torrents:/data
    # Suppression du volume /disk inutile

3. Récupération via rsync depuis les torrents

La majorité des films étaient encore en seed dans Transmission. Récupération par copie directe disque-à-disque :

Prévention

Mesures mises en place

Recycle Bin Radarr
Configuration d'une corbeille avec rétention 7 jours pour éviter toute suppression définitive immédiate.
Single Root Folder
Un seul dossier racine /movies dans Radarr. Suppression des anciens chemins /movies2 obsolètes.
Dozzle
Déploiement d'un agrégateur de logs Docker pour monitoring temps réel de tous les conteneurs.
Audit USB Backup
Vérification et nettoyage de tous les jobs de backup OMV. Séparation stricte disques de stockage / disques de backup.
Retour d'expérience

Leçons apprises

Ne jamais supposer qu'un disque est "libre" sans auditer les services OMV qui y sont attachés. Le plugin USB Backup crée des services systemd qui se déclenchent automatiquement au montage du disque, indépendamment de son usage réel.

Les failsafes de Radarr ont leurs limites. Le message "root folder is empty, rescan will not update as a failsafe" protège partiellement, mais ne suffit pas quand le volume n'est simplement pas monté — Radarr interprète l'absence comme une suppression réelle.

La séparation des rôles est critique en administration système. Un disque ne devrait avoir qu'un rôle unique : stockage OU backup, jamais les deux simultanément sans isolation explicite.
